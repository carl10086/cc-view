Message Types 文档

主要消息类型

USER user

用户发送的消息，包含文本、图片或文件输入。这是会话中最基础的消息类型， 代表用户与 Claude Code 的每次交互输入。 

{
  "type": "user",
  "content": "Hello, can you help me?",
  "timestamp": "2025-01-15T10:30:00Z"
}
ASSISTANT assistant

AI 助手回复的消息，可能包含文本、工具调用或思考过程。这是 Claude 模型 生成的响应，可能包含多步骤推理或代码执行结果。 

{
  "type": "assistant",
  "content": "I'll help you with that...",
  "thinking": true
}
SYSTEM system

系统内部事件，包括信息提示、API 错误、紧凑边界等。System 消息包含 多种子类型，用于记录会话生命周期中的各种系统级事件。 

{
  "type": "system",
  "subtype": "informational",
  "message": "Operation completed"
}
ATTACH attachment

钩子执行结果或技能列表等附加信息。Attachment 消息承载 Hook、Skill 和 Agent 相关的元数据，是会话中重要的结构化数据载体。 

{
  "type": "attachment",
  "subtype": "hook_success",
  "data": { ... }
}
PROMPT last-prompt

最后一次发送给模型的完整提示词。此类型用于记录和调试发送给 Claude API 的完整提示内容，包含所有上下文和指令。 

{
  "type": "last-prompt",
  "prompt": "You are a helpful assistant...",
  "token_count": 2048
}
PERMS permission-mode

权限模式变更记录。当 Claude Code 需要请求用户权限（如文件访问、 命令执行）时，此类型记录权限决策和模式切换事件。 

{
  "type": "permission-mode",
  "mode": "read-only",
  "granted": true
}
FILES file-history-snapshot

文件历史快照，记录某时刻的文件状态。用于追踪文件变更历史， 支持版本回溯和差异对比。 

{
  "type": "file-history-snapshot",
  "files": [
    { "path": "src/index.ts", "hash": "abc123" }
  ]
}
TURNS user-turn

用户对话轮次标记。标识用户主动发起的新一轮对话， 用于会话结构分析和轮次统计。 

{
  "type": "user-turn",
  "turn_index": 5,
  "message_count": 3
}
QUEUE queue-operation

队列操作记录，如任务排队、执行状态。用于追踪异步任务的 生命周期和系统负载情况。 

{
  "type": "queue-operation",
  "operation": "enqueue",
  "task_id": "task-123"
}
TITLE ai-title

AI 生成的会话标题。Claude 根据会话内容自动生成的描述性标题， 用于会话列表展示和快速识别。 

{
  "type": "ai-title",
  "title": "Code Review Session",
  "confidence": 0.95
}

System Message 子类型

System 消息包含多种子类型，用于记录会话生命周期中的各种系统级事件。 以下表格列出了所有已识别的子类型及其说明。

全部子类型 14
Subtype 说明
informational 一般信息提示，如操作完成通知
permission_retry 权限请求重试事件
bridge_status 与外部系统（如 IDE）连接状态变更
scheduled_task_fire 定时任务触发通知
stop_hook_summary Hook 停止执行后的摘要
turn_duration 单轮对话耗时统计
away_summary 用户离开期间的活动摘要
memory_saved 记忆保存确认（如重要信息被记录）
agents_killed Agent 进程被终止通知
api_metrics API 调用指标（token 使用量、延迟等）
local_command 本地命令执行状态
compact_boundary 会话紧凑化边界标记（上下文压缩点）
microcompact_boundary 微紧凑化边界标记
api_error API 调用错误通知
